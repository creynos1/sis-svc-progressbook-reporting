Geocharts

Geocharts are interactive maps that plot data per geographic area. Geocharts use data highlighting, whereby data color-codes areas in varying levels of intensity. Highlighting can affect multiple levels of granularity, to allow for drilling down into regions.

     

A geochart viewed at the country level and the city level

To add a geochart to a report, select the report cell where you want the geochart to appear. Then click the  Geochart Wizard icon. Proceed through the Type, Locations, and Data pages, then click Finish when you are done.

To edit a geochart, double-click it, or select it and click the  Geochart Wizard icon to reopen the wizard.

Type

Choose a Base Map from the list. This selection determines the initial, maximum view level of the geochart. Data from outside this region is not shown on the map.

Set the size of the geochart by specifying fixed Height and Width dimensions, or select the Fit to Cell check box to have the geochart scale to the size of the report cell.

From the Colors list, determine the color range to use for highlighting. Do one of the following:

  • Select a theme. For most themes, lighter highlight colors represent lesser values, and darker colors represent greater values.

  • Choose Linear Range and select a custom color range. The highlight trends towards the first color for lesser values, and towards the second color for greater values.

    If you only want to differentiate values by marker size, choose the same color for both ends of the range.

 Tip
Areas with no data are colored gray.

Select the Show Legend check box to show an explicit scale for the color levels.

Locations

Geocharts support up to three levels of granularity:

Country

Highest level. Select a data cell with country names if the Base Map and data comprises multiple countries.

This level is required if your Region or City names may be ambiguous between countries. For example, "Vancouver" is ambiguous between "Vancouver, British Colombia", and "Vancouver, Washington".

Region

Middle level. This is for country subdivisions such as States, Provinces, or Districts.

This level is required if your City names may be ambiguous between regions. For example, "Portland" is ambiguous between "Portland, Oregon" and "Portland, Maine".

City

Lowest level. Geocharts have no boundary data for cities, so they are shown as circular markers.

In the Show last location type as list, choose how to demarcate the areas at lowest level in your geochart. If country or region is the lowest level, select either circular Markers or geographic Regions. If city is the lowest level, you must select Markers.

Naming scheme

Geographic naming schemes must be consistent within each level. Countries can be named by either their country code abbreviation, English name, or regional name. But do not mix schemes or the application may not interpret the data correctly.

For example, pick one of the following schemes:

  • Germany    | Italy  | United States

  • Deutchland | Italia | United States

  • DEU        | ITA    | USA

  • DE         | IT     | US

Follow similar rules for regions. Cities should be identified by their full names.

Data

Values can be supplied for highlighting and marker size. Each dimension could represent the same data aggregation, two different aggregations of a data set, or two distinct data sets. Data values should be in a Detail or Group section.

Color of Locations must be set.

Size of Markers is optional. If this is not set and the geochart uses markers, Size of Markers uses the Color of Locations field. If the geochart does not use markers, this field is ignored.

For each data dimension to use:

  1. Select a cell to supply the Data Values.

  2. In the Data Labels field, enter a label that will prefix each value when a user hovers over a location.

  3. Choose which aggregate method to use from the Aggregate Type list.

  4. From the Display Format list, choose how the data values are formatted in the legend and hover popup.

  5. From the Decimal Places list, choose a number of decimal places to round the data values.

To use fixed-size markers instead of dynamic size:

  1. Select a report cell containing a fixed numeric value, as written with a preceding equal sign (=). For example: =10.

    Use a value appropriate for the values in the data set.

  2. From the Aggregate Type list, select Avg.
